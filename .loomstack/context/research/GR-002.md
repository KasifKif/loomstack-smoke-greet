Pytest provides several patterns for testing single-function modules efficiently:

*   **Test file naming**: Place tests in `test_*.py` or `*_test.py` files, typically in a `tests/` directory.
*   **Test function naming**: Each test function must start with `test_` and describe what's being verified (e.g., `test_greet_returns_welcome_message`).
*   **Simple assertions**: Use direct `assert` statements for verifying function output, as pytest provides rich assertion introspection.
*   **Parametrized tests**: Use `@pytest.mark.parametrize` to test multiple input/output pairs for the same function logic in one test definition.
*   **Exception testing**: Use `pytest.raises()` context manager to verify that functions raise expected exceptions for invalid inputs.
*   **Fixtures for setup**: Use fixtures to provide common test data or mock objects that multiple tests can reuse.
*   **AAA pattern**: Structure tests as Arrange (setup), Act (call function), Assert (verify results) for clarity.
*   **Test isolation**: Each test should be independent; avoid shared state between tests by using fresh fixtures or data per test.
*   **Edge case coverage**: Test boundary conditions, empty inputs, None values, and other edge cases specific to the function's contract.
*   **Return value verification**: For pure functions, focus tests on verifying the exact return value matches expected output for given inputs.
