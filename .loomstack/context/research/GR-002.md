Pytest offers concise patterns ideal for testing single-function modules:

*   **Basic test functions** — Write `test_<function_name>()` in `test_*.py` files; pytest auto-discovers them.
*   **Direct assertions** — Use standard `assert` statements to validate outputs (e.g., `assert greet("Alice") == "Hello Alice"`).
*   **Fixtures for setup** — Use `@pytest.fixture` to create reusable setup/teardown (e.g., mock data, environment config).
*   **Parametrized tests** — Use `@pytest.mark.parametrize` to run the same test logic with multiple input/output pairs.
*   **Exception testing** — Wrap code expected to raise with `pytest.raises(ExpectedException)` to assert exceptions.
*   **Monkeypatching** — Use the `monkeypatch` fixture to temporarily replace attributes, environment variables, or functions for isolation.
*   **Temporary filesystem** — Use `tmp_path` fixture to create isolated temp files/directories for file-based tests.
*   **Simple class-based grouping** — Group related tests in `class Test<Feature>` if multiple tests exist for one function.
*   **Test discovery** — Name test files `test_*.py` or `*_test.py`, and functions `test_*` to ensure automatic discovery.
*   **No boilerplate** — Avoid `unittest.TestCase` subclassing unless needed; plain functions and classes work out of the box.
